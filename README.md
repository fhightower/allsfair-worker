# allsfair-worker

Production serving stack for [Allsfair](https://github.com/fhightower/allsfair),
a game of simultaneous, predetermined moves. Live at
[allsfair.hightower.space](https://allsfair.hightower.space/).

One Cloudflare Worker serves everything:

- `GET /*` — static frontend (`public/index.html`)
- `POST /api` — game API (`create_game`, `join_game`, `submit_move`, `get_moves`)
- D1 (SQLite) stores games and moves; board state is derived by replaying the
  move log

## Relationship to the Python engine

The [`allsfair`](https://github.com/fhightower/allsfair) Python package remains
the source of truth for game rules; `src/engine.ts` is a byte-faithful port.
`test/parity.test.ts` replays fixtures generated by the Python engine
(`scripts/generate_parity_fixtures.py` in that repo) and asserts identical
board states and HTML output. After any rules change in Python, regenerate the
fixtures and fix this port until parity passes.

The ML bot is not wired up yet (`create_game` with `play_against_ml` returns
400). The seam for re-adding it is documented in
`docs/superpowers/specs/2026-07-06-cloudflare-migration-design.md`.

## Development

```shell
npm install
npm test        # vitest; D1 is stubbed with node:sqlite (see test/d1-shim.ts)
npx tsc         # typecheck
```

Note: `wrangler dev` and miniflare need workerd, which requires macOS 13.5+ or
Linux. On older machines the test suite still runs (plain node), and real-runtime
verification happens against the deployed Worker.

## Deploy

```shell
npx wrangler deploy
```

Schema changes go in `migrations/`, applied with:

```shell
npx wrangler d1 migrations apply allsfair --remote
```
